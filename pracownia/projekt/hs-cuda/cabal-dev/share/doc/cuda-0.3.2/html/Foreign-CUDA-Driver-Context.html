create :: Device -> [ContextFlag] -> IO Context
Create a new CUDA context and associate it with the calling thread
attach :: Context -> [ContextFlag] -> IO ()
Increments the usage count of the context. API: no context flags are currently supported, so this parameter must be empty.
detach :: Context -> IO ()
Detach the context, and destroy if no longer used
destroy :: Context -> IO ()
Destroy the specified context. This fails if the context is more than a single attachment (including that from initial creation).
current :: IO Device
Return the device of the currently active context
pop :: IO Context
Pop the current CUDA context from the CPU thread. The context must have a single usage count (matching calls to attach/detach). If successful, the new context is returned, and the old may be attached to a different CPU.
push :: Context -> IO ()
Push the given context onto the CPU's thread stack of current contexts. The context must be floating (via pop), i.e. not attached to any thread.
sync :: IO ()
Block until the device has completed all preceding requests
getLimit :: Limit -> IO Int
Query compute 2.0 call stack limits
setLimit :: Limit -> Int -> IO ()
Specify the size of the call stack, for compute 2.0 devices
setCacheConfig :: Cache -> IO ()
On devices where the L1 cache and shared memory use the same hardware resources, this sets the preferred cache configuration for the current context. This is only a preference.
